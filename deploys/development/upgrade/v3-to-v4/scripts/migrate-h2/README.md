# Spring Batch Migration Scripts: H2 to PostgreSQL

This folder contains scripts to migrate Spring Batch metadata from an H2 database file to a PostgreSQL instance. The process uses Docker containers to run the necessary tools (Java/H2 for extraction and PostgreSQL/psql for injection), ensuring no local dependencies are required other than Docker.

## Prerequisites

- Docker installed and running.
- Execution permissions for the scripts:
  ```bash
  chmod +x export.sh import.sh
  ```

## Migration Workflow

### 1. Export Data from H2
The `export.sh` script extracts data from an H2 `.mv.db` file, cleans the syntax for PostgreSQL compatibility, and generates a standalone SQL file.

**Usage:**
```bash
./export.sh <path_to_h2_file>
```
- `<path_to_h2_file>`: Full path to the `.mv.db` file.
- **Output:** Generates `pg_data_only.sql` in the current directory.

### 2. Import Data to PostgreSQL
The `import.sh` script connects to a running PostgreSQL instance, temporarily disables foreign key constraints, inserts the data, and updates the Spring Batch sequences to prevent ID collisions.

**Usage:**
```bash
./import.sh <input_sql> <pg_host> <pg_port> <pg_user> <pg_password> <pg_db>
```
- `input_sql`: The file generated by the export script (e.g., `pg_data_only.sql`).
- `pg_host`: IP or hostname of the PostgreSQL server.
- `pg_port`: Port (default is 5432).
- `pg_user`: Database username.
- `pg_password`: Database password.
- `pg_db`: Target database name.

## Post-Migration Verification

### Database Level
Verify that record counts match between the old and new environments:
```sql
SELECT COUNT(*) FROM batch_job_instance;
SELECT COUNT(*) FROM batch_job_execution;
```

### Application Level
1. Check application logs during startup for any `PSQLException` or sequence-related errors.
2. Verify that the historical job data is visible in the application UI/API.

### Execution Test
The most critical test is running a new Job. If the sequences were updated correctly, the application will successfully create new records without "duplicate key" errors.
